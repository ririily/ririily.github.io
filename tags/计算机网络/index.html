网络是如何连接的 读书记录

网络是如何连接的 读书记录 2 用电信号传输TCP/IP数据 协议栈内部结构 应用程序会将收发数据等工作委派给下层的部分来完成，下面是 Socket 库，其中包括解析器，解析器用来向 DNS 服务器发出查询 浏览器、邮件等一般应用程序收发数据时用 TCP； DNS 查询等收发较短的控制数据时用 UDP。 然后是用 IP 协议控制网络包收发操作的部分。在互联网上传送数 据时，数据会被切分成一个一个的网络包 A，而将网络包发送给通信对象的操作就是由 IP 来负责的。此外，IP 中还包括 ICMP 协议和 ARPB协议。 ICMP 用于告知网络包传送过程中产生的错误以及各种控制消息，ARP 用 于根据 IP 地址查询相应的以太网 MAC 地址 。 IP 下面的网卡驱动程序负责控制网卡硬件，而最下面的网卡则负责完 成实际的收发操作，也就是对网线中的信号执行发送和接收的操作。 协议栈是根据套接字中记录的控制信息来工作的。 创建套接字时，首先分配一个套接字所需的内存空间，然后向其 中写入初始状态。 控制信息分为两类 数据包头部信息 套接字中记录的用来控制协议操作的信息 套接字和描述符 套接字接口为网络通信增加了一个新的抽象，即套接字。和文件一样，每个活动的套接字都有一个整数来标识，我们将其称为套接字描述符。 connect（< 描述符 >, < 服务器 IP 地址和端口号 >, …） 上面的调用提供了服务器的 IP 地址和端口号，这些信息会传递给协议 栈中的 TCP 模块。然后，TCP 模块会与该 IP 地址对应的对象，也就是与 服务器的 TCP 模块交换控制信息 连接操作的第一步是在 TCP 模块处创建表示连接控制信息的头部。 通过 TCP 头部中的发送方和接收方端口号可以找到要连接的套 接字。
